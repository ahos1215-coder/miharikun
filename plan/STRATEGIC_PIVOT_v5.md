## 2. 廃止事項：資格管理フック

Blueprint v4 の「資格管理＝無料フック → 規制マッチング＝有料」モデルを **完全廃止**。

### 廃止理由

1. 利用者は既に別の手段で資格管理しており、本サービスで再入力する魅力・メリットが薄い
2. 5年に1度の更新頻度よりも、日々発生する「自分に関係ある情報のフィルタリング」の方が圧倒的に需要が高い

### 影響する v4 セクション

| v4 セクション | 変更 |
|--------------|------|
| §2 競合分析「資格管理が入口」 | 廃止。入口は「自船プロファイル登録 → 即マッチング」 |
| §6.2 乗組員プロファイル | 証明書期限管理を削除。職種・航行区域のみ残す |
| §8.1 MVP ルート `/crew/certificates` | 削除 |
| §9 ビジネスモデル Free ティア「資格登録」 | 削除。Free は「ニュース閲覧のみ」 |
| §13 Phase 2「証明書の期限管理」 | 削除 |

## 3. 技術的アプローチ：Self-hosted Runner 採用

ClassNK の IP ブロック問題を解決するため、**開発環境 PC を GitHub Self-hosted Runner として設定**する。

### 構成

```
GitHub Actions (ubuntu-latest)
  └── MLIT RSS / MLIT Crawl / process-queue / health-check

GitHub Actions (self-hosted, Windows)
  └── NK Daily Scraper（ローカル回線経由でアクセス）
```

### メリット

- IP ブロックを回避しつつスケジュール通りの自動収集を維持
- 追加コストなし
- GHA の `runs-on: self-hosted` に変更するだけで移行可能

## 4. 開発優先順位の再構成

### 最優先（Phase 1 R2 → Phase 2 で連続実装）

#### A. 詳細な Ship Specs 登録

マッチング精度に直結するプロファイル機能。以下のフィールドを **必須** で登録させる：

| フィールド | 型 | マッチング用途 |
|-----------|-----|--------------|
| ship_name | TEXT | 表示用 |
| ship_type | ENUM | 船種別規制の適用判定 |
| gross_tonnage | INT | GT 閾値規制（SOLAS, MARPOL 等） |
| dwt | INT | 載貨重量トン数規制 |
| build_year | INT | 建造年規制（EEDI, Tier III 等） |
| classification_society | ENUM | NK/JG/ABS/DNV/LR/BV |
| flag_state | TEXT | 旗国別規制 |
| navigation_area | ENUM[] | 国際/近海/沿海/平水 |
| routes | TEXT[] | 具体的航路（「東京湾→シンガポール」等） |
| imo_number | TEXT | IMO 番号（任意、将来の自動照合用） |

#### B. 高度なマッチングエンジン

Gemini を活用し、PDF の内容と自船スペックを照合して「適用/非適用」を判定。

```
入力: regulation (PDF 分類結果) × ship_profile (ユーザー登録)
出力: {
  "applicable": true/false,
  "reason": "GT 500 以上の国際航行船舶に適用（本船 GT 2,800）",
  "confidence": 0.95,
  "citations": [{"text": "本規則は総トン数500以上の...", "page": 2}]
}
```

**マッチングの3段階:**

1. **ルールベース（高速）**: GT 範囲、船種、建造年の単純比較 → 明確に非該当を除外
2. **AI マッチング（精密）**: ルールベースで「可能性あり」の規制を Gemini に送り、自船スペックとの照合を依頼
3. **手動レビュー（フォールバック）**: confidence < 0.7 の場合はユーザーに「要確認」として表示

#### C. 超軽量通知 UI

船上の低帯域環境（VSAT、衛星通信）でも「自分に関係のある1件」の要約と根拠を最速で確認できる UI/UX。

**設計原則:**

- 1ページ完結（ダッシュボードに全情報を集約）
- 初期ロード < 50KB（HTML + 最小 CSS、JS は遅延読み込み）
- 画像なし、アイコンはテキスト絵文字
- オフライン対応（Service Worker でキャッシュ）
- メール1通で完結する週次サマリー（HTML メール、画像なし）
